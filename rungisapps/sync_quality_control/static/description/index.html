Quality Control

Only impeccable raw materials and semi-finished products that fulfill defined quality requirements may be used in the production process. Therefore, high inspection standards are needed for incoming goods.

To reduce non-conformance costs, a comprehensive inspection of incoming goods is important to verify if the material arrived in good condition at your warehouse. The goods will be accepted in the stock through systematic incoming inspection, based on business rules which comply with existing business processes and regulations.

This application provides functionality for quality inspections on incoming product's (Receipts). It can be configured with different control points and its parameters product wise. It also provides quality alert report with/without barcode and product lot number wise. The quality issues caused by suppliers are identified before they reach your production, not later when it's too late.

Key Features

  • Quality control point is defined for receipt's order which will generate multiple quality inspections on incoming product (Receipts).
  • It could be configured with different possibilities, measurement (min/max tolerance), image, pass or fail option (choice) with multiple parameters for one control point.
  • System will reconfirm user input data and suggest right measurement, when wrong data is fetched by user.
  • Quality alerts stages will be defined.
  • Multiple quality alerts can be generate with multiple reasons for quality inspection.
  • Alert email related to quality issues could be sent to quality alert team and responsible person.
  • Quality alert sticker report will be generated with/without product barcode and product lot number to track the products physically for repairs.
  • Restricted quality check on picking operation as per configuration on quality control point.

Configuration of Quality Control

Under configuration menu create quality control alert team.

Dashboard

Quality alerts and Quality Inspections can be directly redirected to different teams' dashboard.

Quality Control Points

Different quality control points can be configured based on products and operations. Same product multiple control points versions and sequences can be managed.
Creation time of quality control point if restrict checked than if user try to done particular operation which mention here without check inspection or inspection status is not passed then systeam raise warning.
We can configure below different 'Test Type' in control points.
  • Measure (min/max tolerance)
  • Image
  • Choice (Pass or Fail)
On pressing 'Quality Inspections' button, inspection will begin based in configuration for incoming product.
System will suggest right measurement parameters as configured during inspection.
'QCP00001' is a auto generated sequential number, which represent product 'Quality control point'.
'QCPL00001' is a auto generated sequential number, which represent control point perameters 'Quality control point line'.

Quality Inspections

Quality Inspections can be used with group by functionality to identify records inspection wise.
'QI00005' is a auto generated sequential number, which represent product 'Quality Inspection'.
'IL00029' is a auto generated sequential number, which represent tested quality insepction perameter 'Inspection Line'.

Quality Alerts

Quality alerts with multiple reasons can be generated for failed inspection line.
Quality alert sticker report will be generated with/without product barcode and product lot number to track the products physically for repairs.
'QA00001' is a auto generated sequential number, which represent product 'Quality alert'.

Restrict Picking

If you configure "Is Restrict" on quality control point then system will be not allowed to submit the picking without checking quality.

Detailed Quality Inspection

If you configure "Detailed Quality Inspection" on quality control point then system will be check quality control by detailed operations wise.
